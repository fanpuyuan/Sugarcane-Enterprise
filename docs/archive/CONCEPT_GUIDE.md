# 核心概念通俗指南 (The "Explain Like I'm 5" Guide)

您提到对这些概念感到陌生，这完全没问题。在现代软件开发中，它们就像建筑工地上的**图纸、仓库、集装箱和流水线**。

## 1. Git：时光机 (Time Machine)
**它是做什么的？**
想象您在写 Word 文档时，经常会“另存为”：`论文_初稿.doc`、`论文_修改版.doc`、`论文_最终版.doc`、`论文_打死不改版.doc`。
Git 就是一个超级智能的“另存为”系统。

*   **存档 (Commit)**：每次您觉得代码写得差不多了，点一下“提交”，Git 就会给当前所有文件拍个“快照”。
*   **回滚 (Revert)**：如果今天改乱了，您可以瞬间通过时光机回到昨天的状态。
*   **分支 (Branch)**：想象平行宇宙。您可以在“平行宇宙A”里大胆尝试新功能，如果搞砸了，直接把这个宇宙炸掉，主宇宙（主线代码）丝毫不受影响。

**我们刚才做了什么？**
执行 `git init`，就是给您的文件夹装上了这个“时光机”。

## 2. GitHub：云盘 / 多人联机服务器
**它是做什么的？**
如果 Git 是您电脑上的单机游戏存档，GitHub 就是云端服务器。

*   **备份 (Remote)**：把您电脑上的代码推送到 GitHub，借用微软的服务器做异地备份。电脑炸了，代码还在。
*   **协作 (Collab)**：团队成员可以把代码传上去，GitHub 会帮你们合并。比如您写后端，您朋友写前端，GitHub 负责把两部分拼在一起。

**我们刚才做了什么？**
我建议您去网站上创建仓库，就是为了准备好这个“云端基地”。

## 3. Docker：神奇集装箱 (Magic Container)
**它是做什么的？**
这是最天才的发明。
**以前的痛点**：我在我电脑上装好了数据库，发给您，您电脑上死活跑不起来（缺文件、版本不对、系统冲突...）。这叫“在我电脑上是好的”。
**Docker 的解法**：
不管您要用数据库、还是 AI 模型，我都把它打包进一个**集装箱 (Image/镜像)** 里。这个集装箱里有它运行需要的一切（系统环境、依赖库、配置文件）。
您只需要在电脑上装一个“码头” (Docker Desktop)，把集装箱拉下来，插上电就能跑。

*   **主要好处**：不用配置环境！
    *   以前装 PostGIS 数据库：下载安装包、配环境变量、改系统配置……折腾半天。
    *   **现在**：`docker run`。一分钟搞定。

**我们刚才做了什么？**
`docker-compose.yml` 就是一张**货运清单**。
我们告诉 Docker：“我要一个 PostGIS 集装箱，要一个 MinIO 集装箱，还要一个 AI 集装箱...”。Docker 就会自动去港口（Docker Hub）把它们拉回来并在您电脑上跑起来。
刚才那几分钟的下载，就是它在搬运这些“家具齐全的房间”。

## 4. CI/CD：全自动机器人流水线
*   **CI (Continuous Integration - 持续集成)**：
    这里的关键是**自动测试**。
    每当您把代码传到 GitHub，就会触发一个机器人。它会自动帮您运行所有测试代码。
    *比喻*：您写完作业交上去，机器人老师立刻批改，如果有错别字（Bug），马上打回让您重写，不准入库。

*   **CD (Continuous Delivery - 持续交付)**：
    这里的关键是**自动发布**。
    代码测试通过后，机器人自动把它打包、发送到生产服务器上更新软件。
    *比喻*：作业批改通过后，机器人自动把它打印出来，分发给全班同学（用户），不需要您亲自去发。

**目前阶段**：
在项目初期（现在），我们暂时还不需要复杂的 CI/CD。我们现在主要是在本地（Local）造车。等以后这辆车要量产上市了，我们再搭建流水线。

---

## 总结：简单理解
1.  **Git**：防后悔药（本地存档）。
2.  **GitHub**：云端仓库（远程备份 + 找人帮忙）。
3.  **Docker**：预装家具的样板房（不折腾环境，拿来即用）。
4.  **CI/CD**：以后才需要的全自动工厂。

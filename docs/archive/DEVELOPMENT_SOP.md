# 标准开发流程 (Standard Operating Procedure)

为了确保项目长期稳定可维护，我们将采用一线大厂通用的 **"Design First" (设计先行)** 开发模式。

## 📅 总体开发流向

1.  **产品设计 (PRD)** ✅ *[已完成]*
    *   明确做什么、不做什么。
    *   成果：`PROJECT_PLAN.md`, `SYSTEM_FLOW.md`

2.  **交互与视觉设计 (UED)** ⬅️ **我们在这里**
    *   决定长什么样。
    *   工具：Figma / Sketch。
    *   成果：高保真原型图 (UI Spec)。

3.  **技术方案设计 (Tech Spec)**
    *   决定怎么实现。
    *   **数据库设计**: ER 图, SQL DDL。
    *   **接口定义**: OpenAPI (Swagger), 前后端按照这个"契约"开发。
    *   成果：`schema.sql`, `api.yaml`。

4.  **编码实施 (Coding)**
    *   后端 (Java): 实现 API 逻辑。
    *   前端 (Android): 还原 UI, 对接 API。
    *   AI (Python): 训练/封装模型。

5.  **测试与联调 (QA & Integration)**
    *   单元测试 (Unit Test)。
    *   链路联调 (End-to-End)。

---

## 🚀 为什么必须先做 UI 设计 (Step 2)？

在没有 Figma 之前，**严禁写前端代码**。因为：
1.  **避免反复横跳**：UI 定了，字段才能定；字段定了，数据库才能设计。
2.  **视觉标准**：前端开发需要从 Figma 直接复制 CSS 代码（颜色、间距、字号），而不是靠肉眼猜。
3.  **资源提取**：所有的图标 (Icons)、切图 (Images) 都需要从 Figma 导出。

---

## 🤖 AI 提速方案 (AI Acceleration)

在大厂流程中，AI 可以在这些环节提速：

*   **UI 阶段**: 您负责画线框和布局，AI (我) 可以帮您写具体组件的样式代码，或者生成用来填充界面的文案/图片。
*   **Tech Spec 阶段**: **(重度使用)** 您只要把 UI 截图给我，我可以自动帮您推导出数据库表结构 (SQL) 和 API 接口定义。
*   **Coding 阶段**: **(重度使用)** 我可以根据 API 定义，直接生成 Java Controller 代码和 Android Retrofit 客户端代码。
